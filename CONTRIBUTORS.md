# Contributors Guide

Thank you for your interest in contributing to Talos Tools! This document outlines the process for becoming a contributor and the guidelines you should follow.

## Access Policy

Please note that this is a **private repository**. Unsolicited pull requests are not possible as write access is restricted to approved contributors only.

## Contribution Guidelines

To maintain code quality and consistency, please adhere to the following guidelines:

### 1. Code Style
- Follow the existing code style and formatting conventions (Prettier/ESLint).
- Use TypeScript for type safety whenever possible.
- Ensure your code is clean, readable, and well-commented where necessary.

### 2. Workflow
- **Branching**: Create a new branch for each feature or bug fix. Use descriptive names like `feature/new-tool` or `fix/login-issue`.
- **Commits**: Write clear and concise commit messages. Start with a verb (e.g., "Add...", "Fix...", "Update...").
- **Pull Requests**: When your changes are ready, submit a Pull Request (PR) against the `main` branch. Provide a clear description of what changed and why.

### 3. Testing
- Verify your changes locally before pushing.
- Ensure that you haven't introduced any regression bugs.
- If adding significant new functionality, please include relevant tests or testing steps in your PR description.

### 4. Communication
- If you are unsure about an implementation detail, reach out to the team before spending too much time on it.
- Be respectful and constructive in code reviews and discussions.

Thank you for helping us build Talos Tools!
